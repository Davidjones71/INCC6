These tests are performed at the end of each measurement cycle. If there is exact agreement, the tests pass; otherwise, an error message is generated and the cycle data are not further processed. Failure of the checksum tests indicates an electronic failure unless the multiplicity distributions have multiplicities above 127, in which case the checksum tests can be turned off to allow conventional coincidence counting with multiplicity electronics.
